# 1. Introduction

User personas represent the different types of users who interact with NEXORA AI.

Understanding users helps us design:

- Better features
- Better workflows
- Better AI experiences
- Better business decisions


NEXORA AI has four primary user categories:

1. Students
2. Recruiters
3. Colleges
4. Platform Administrators


# 2. Persona 01 — Student


## Persona Name

Future Builder


## User Type

Student / Job Seeker


## Description

A college student who wants to build skills, discover career opportunities, and get hired based on actual capability rather than only academic records or resumes.


## Goals

- Find the right career path
- Understand current skill level
- Improve technical abilities
- Build strong projects
- Get relevant job opportunities
- Showcase real achievements


## Challenges

- Confusion about what skills to learn
- Lack of personalized guidance
- Difficulty understanding industry expectations
- Weak resume presentation
- Difficulty finding suitable opportunities


## Current Behavior

Students usually:

- Watch random courses
- Follow online tutorials
- Apply through job portals
- Prepare without clear direction


## Needs From NEXORA AI

Student needs:

- AI career mentor
- Skill analysis
- Personalized roadmap
- Resume improvement
- Project evaluation
- Job matching


## Success Feeling

"I finally know what I need to learn and where I can reach."

# 3. Persona 02 — Recruiter


## Persona Name

Talent Hunter


## User Type

Company Recruiter / Hiring Manager


## Description

A professional responsible for finding skilled candidates and building company teams.


## Goals

- Find qualified candidates quickly
- Reduce hiring time
- Identify hidden talent
- Verify candidate skills
- Improve hiring quality


## Challenges

- Too many applications
- Fake or exaggerated skills
- Difficult resume screening
- Limited technical evaluation
- Slow recruitment process


## Current Behavior

Recruiters usually:

- Post jobs
- Filter resumes manually
- Conduct interviews
- Depend on experience and keywords


## Needs From NEXORA AI

Recruiters need:

- AI candidate discovery
- Skill-based search
- Candidate ranking
- Job requirement analysis
- Talent intelligence reports


## Success Feeling

"I found the right candidate without wasting time."

# 4. Persona 03 — College Administrator


## Persona Name

Career Coordinator


## User Type

College Placement Team


## Description

A person responsible for managing student placement activities and improving employment outcomes.


## Goals

- Increase placement rate
- Track student readiness
- Connect with companies
- Improve training programs


## Challenges

- Manual student tracking
- Lack of skill visibility
- Difficulty identifying weak areas
- Limited recruiter connections
- Data scattered across systems


## Current Behavior

Colleges usually:

- Maintain spreadsheets
- Conduct training programs
- Collect placement data manually


## Needs From NEXORA AI

College needs:

- Student analytics dashboard
- Skill gap reports
- Placement insights
- Recruiter management
- Performance tracking


## Success Feeling

"We know exactly how prepared our students are."

# 5. Persona 04 — Platform Administrator


## Persona Name

System Controller


## User Type

NEXORA AI Admin


## Description

Responsible for maintaining platform operations, security, and overall system health.


## Goals

- Maintain platform stability
- Manage users
- Monitor activities
- Ensure security


## Challenges

- Managing large user data
- Monitoring system performance
- Preventing misuse
- Maintaining compliance


## Needs From NEXORA AI

Admin requires:

- User management
- Analytics dashboard
- Security controls
- System monitoring
- Data management


## Success Feeling

"The platform operates securely and efficiently."
